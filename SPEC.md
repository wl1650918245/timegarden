## 1. 产品概述

### 1.1 定位
轻量化桌面事项管理工具，聚焦当周最重要事项的进度监督。通过植物生长/枯萎的视觉反馈 + 花瓣进度环 + 渐变色卡片，提供直观的压力感知和完成成就感。

### 1.2 目标用户
- 需要高频查看周计划的职场人士
- 追求美观和氛围感而非功能堆叠的效率工具用户

### 1.3 核心价值
| 维度 | 描述 |
|------|------|
| 存在感 | 三视图体系：Nano 悬浮花(48×68) → Mini 花盆(可缩放) → Full 面板(可缩放) |
| 紧迫感 | 卡片渐变色 + 紧迫度标签随截止日临近变化，植物随任务进度枯萎/盛开 |
| 满足感 | 完成时粒子特效 + 开花 + 花瓣环进度更新 |
| 专注力 | 内置专注计时器，记录每项任务花费时间，Nano 专注条自适应任务名宽度 |
| 低干扰 | Nano 最小仅 48×68，悬浮于桌面边缘 |

### 1.4 三视图体系
| 视图 | 尺寸 | 触发 |
|------|------|------|
| Nano 悬浮花 | 48×68 | 从 Mini 收起、专注模式 |
| Mini 花盆 | 360×310（可缩放 ≥280×240） | 默认启动 |
| Full 面板 | 720×520（可缩放 ≥480×360） | 展开操作 |

### 1.5 双主题
| 主题 | 文件 | 风格 |
|------|------|------|
| 暗色 Dark Glass | style.css | 深色玻璃 + 翡翠绿强调 |
| 亮色 Warm Ceramic | style-old.css | 日式陶器温暖极简 |

### 3.4 任务数据结构

```typescript
interface Task {
  id: string;            // 唯一标识
  title: string;         // 任务标题
  completed: boolean;    // 是否完成
  priority: 'high' | 'medium' | 'low';  // 手动优先级（默认 medium）
  deadline: string | null;  // 截止日 "YYYY-MM-DD" 或 null(无截止日)
  createdAt: string;     // ISO 创建时间
  weeklyTimeMs: number;  // 本周专注累计毫秒
}

interface WeekData {
  weekKey: string;       // "2026-W23"
  important: Task[];     // 重要事项 (最多4条)
  todos: Task[];         // 待办列表 (不限数量)
}
```

### 3.5 紧迫度与优先级系统

#### 自动紧迫度（基于截止日）
```typescript
function calcUrgency(task: Task): { days: number; label: string; color: string } {
  if (task.completed) return { days: 99, label: '已完成✓', color: 'done' };
  if (!task.deadline) return { days: -1, label: '无截止日', color: 'none' };

  const today = new Date();
  const deadline = new Date(task.deadline);
  const diff = Math.ceil((deadline.getTime() - today.getTime()) / (1000*60*60*24));

  if (diff >= 3) return { days: diff, label: `剩${diff}天`, color: 'green' };
  if (diff === 2) return { days: 2, label: '剩2天', color: 'yellow' };
  if (diff === 1) return { days: 1, label: '⚠明天截止', color: 'orange' };
  if (diff <= 0) return { days: 0, label: '🔥过期!', color: 'red' };
}
```

#### 手动优先级
| 优先级 | 图标 | 默认值 | 设置入口 |
|--------|------|--------|----------|
| 高 priority.high | 🔴 | - | 新增/编辑时选择 |
| 中 priority.medium | 🟡 | ✅ 默认 | 新增/编辑时选择 |
| 低 priority.low | 🟢 | - | 新增/编辑时选择 |

#### 视觉叠加规则
- 收起态：显示紧迫度颜色（渐变色卡片）
- 展开态要事：左色条 = 紧迫度色 + 卡片渐变色
- 展开态待办：圆形复选框边框颜色 = 优先级色
- 待办列表顶部显示高优/中优计数

## 4. 专注计时系统

### 4.1 进入专注
- Mini/Full 任务卡片点击 ⏱ 按钮 → 开始专注
- 自动切到 Nano 视图，窗口宽度自适应任务名长度（180~440px）
- Nano 计时条：⏱ 图标 + 任务名 + 实时计时器（<1h: m:ss, ≥1h: h:mm:ss）
- 状态圆点绿色脉冲动画
- 专注中 hover 和右键菜单正常可用

### 4.2 退出专注
- 点击计时条 ⏱ 或点击花朵 → 停止专注
- 累计时间写入 `task.weeklyTimeMs`
- Nano 窗口缩回 48×68（普通花态）

### 4.3 跨视图专注
- 专注中可切换到 Mini/Full → 计时器继续，任务卡片显示累计时间
- 停止专注时：若在 Nano 模式则缩窗，若在 Mini/Full 则保持当前尺寸

### 4.4 25 分钟循环进度条
- 计时条底部 `::before` 伪元素从左侧开始填充，`scaleX` 从 0→1
- 填充色：亮绿渐变（暗色 `rgba(74,222,128,0.15→0.25)`，亮色 `rgba(90,172,64,0.12→0.22)`）
- 每 25 分钟自动重置，开始下一个循环（`FOCUS_CYCLE_MS = 25 × 60 × 1000`）
- 进度通过 CSS 变量 `--focus-progress` 驱动，每秒更新，`transition: transform 0.8s linear` 平滑过渡

### 4.5 最后 5 分钟预警
- 剩余 ≤5 分钟时，计时条添加 `.focus-warning` class
- 边框从绿色变为琥珀色（暗色 `rgba(245,158,66,0.4)`，亮色 `rgba(200,140,60,0.4)`）
- 呼吸动画加速：从 3s 周期切换到 1.5s 周期（`focusBreathFast` keyframe）
- 预警在下一循环开始时自动解除

### 4.6 专注动效
| 动效 | 周期 | 实现 |
|------|------|------|
| 呼吸光晕 | 3s | `focusBreath` keyframe：`border-color` + `box-shadow` 明暗循环 |
| 专注流光 | 4s | `::after` 对角渐变 `translateX(-100%→100%)` |
| 计时器脉冲 | 2s | `timerPulse` keyframe：`opacity` + `text-shadow` 起伏 |
| ⏱ 图标轻弹 | 6s | `focusIconSpin` keyframe：88% 处 `scale(1.25)` |
| 状态圆点脉冲 | 1.2s | `focusPulse` keyframe：`box-shadow` + `scale` 呼吸 |
